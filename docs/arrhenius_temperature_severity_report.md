# 基于 Arrhenius 简化模型的全球高温严苛度对比报告

生成日期：2026-05-07

## 结论摘要

如果评价目标是电子器件寿命中常见的“温度每升高 10°C，寿命约减半”这一工程近似，那么不应只比较年平均温度，而应对逐小时温度做指数加权。

在本次候选热城样本中，按 NASA POWER `2016-2025` 小时级 `T2M` 数据计算：

- **科威特市不是最严苛点位**。
- **阿瓦士、雅各布阿巴德、巴士拉、吉达、加奥、吉赞等点位的 Arrhenius 等效严苛度高于科威特市**。
- **吉赞的年平均气温最高，但极端高温小时不多；按 10°C 寿命减半模型加权后，仍比科威特市严苛约 10%**。
- **苏丹港年平均气温高于科威特市，但高温尾部较弱；按指数加权后与科威特市几乎打平**。

可引用表述：

> 如果采用电子可靠性中常见的 Arrhenius 温度加速近似，即温度每升高 10°C 寿命约减半，则全球严苛度不能只看年平均温度，而应按小时温度进行指数加权。本次基于 NASA POWER 2016-2025 小时级 `T2M` 的候选热城对比显示，阿瓦士、雅各布阿巴德、巴士拉、吉达、加奥、吉赞等点位均比科威特市更严苛；吉赞相对科威特市的温度损伤指数约为 `1.10x`，苏丹港约为 `1.00x`。

## 1. 理论依据

电子器件可靠性中，温度对寿命的加速作用通常用 **Arrhenius 温度加速模型**描述。其常见形式为：

```text
AF = exp[(Ea / k) * (1 / T_use - 1 / T_stress)]
```

其中：

- `AF`：温度加速因子
- `Ea`：激活能，单位通常为 `eV`
- `k`：Boltzmann 常数
- `T_use` / `T_stress`：使用温度 / 应力温度，必须用 Kelvin

“温度每升高 `10°C`，寿命减半”不是严格物理常数，而是 Arrhenius 模型在工程沟通中的简化近似，常被称为 **10°C rule** 或 **Q10 rule**。它适合做初筛和相对比较，但精确寿命预测仍应使用具体器件、具体失效机制对应的 `Ea`、结温 `Tj` 和厂家可靠性数据。

参考来源：

- NIST Engineering Statistics Handbook: Arrhenius 模型用于温度加速与可靠性分析，且激活能依失效机制而变。[NIST Arrhenius](https://www.itl.nist.gov/div898/handbook/apr/section1/apr151.htm)
- Texas Instruments 可靠性应用说明：Arrhenius 模型常用于半导体可靠性预测和温度加速因子计算。[TI SLVA118A](https://www.ti.com/lit/pdf/slva118)

## 2. 本报告计算口径

### 数据源

- 数据源：NASA POWER Hourly API
- Endpoint：`https://power.larc.nasa.gov/api/temporal/hourly/point`
- 参数：`T2M`
- 含义：2 米气温小时平均值，单位 `°C`
- 时间标准：`LST`，Local Solar Time
- 时间范围：`2016-01-01 00:00` 至 `2025-12-31 23:00`
- 每个完整点位小时数：`87,672`

NASA POWER Hourly API 支持小时级参数请求，并支持 `time-standard=LST` 或 `time-standard=UTC`。[NASA POWER Hourly API](https://power.larc.nasa.gov/docs/services/api/temporal/hourly/)

### 指标定义

若采用“每升高 `10°C`，损伤速率翻倍”的简化模型，则每小时相对损伤可写为：

```text
damage_rate(T) = 2^(T / 10)
```

对一个城市的全部小时温度序列求平均：

```text
damage_index = mean(2^(T2M_C / 10))
```

再换算成一个更容易理解的“等效恒定温度”：

```text
T_eq = 10 * log2(damage_index)
```

解释：

- `T_eq` 越高，说明按 Arrhenius 简化规则的长期温度损伤越严苛。
- `damage_ratio_vs_kuwait_city` 表示相对科威特市的温度损伤倍率。
- 如果设备内部温升 `DeltaT = Tj - Tamb` 对所有地区相同，则排名不变，因为所有点位的损伤指数都会乘以同一个 `2^(DeltaT/10)`。

## 4. 关键观察

### 4.1 科威特市不是最严苛温度点

科威特市的极端高温很强，最高小时 `T2M` 达到 `48.49°C`，`>=40°C` 小时数为 `7828`。但它的冬季低温也明显，10 年平均 `T2M` 只有 `26.97°C`。因此按全生命周期小时积分后，它不是本批样本中最严苛点位。

### 4.2 阿瓦士、巴士拉这类地点更受极端高温尾部支配

阿瓦士、巴士拉虽然年平均温度低于吉赞，但高温尾部极强：

- 阿瓦士：`>=45°C` 小时 `8114`，P99 `50.39°C`
- 巴士拉：`>=45°C` 小时 `6009`，P99 `48.85°C`

在 `2^(T/10)` 指数加权下，这类高温尾部会显著推高损伤指数。

### 4.3 吉赞是“长期持续热”，不是“极端峰值热”

吉赞的 10 年平均 `T2M` 为 `30.93°C`，在本批样本中最高。但它 `>=40°C` 小时只有 `45`，`>=45°C` 小时为 `0`。这说明吉赞的严苛性来自全年持续高温和低冷却季，而不是极端峰值。

因此：

- 用年平均温度看，吉赞非常突出。
- 用 Arrhenius 指数损伤看，吉赞仍比科威特市严苛，但不如阿瓦士、雅各布阿巴德、巴士拉等极端高温尾部更重的地点。

### 4.4 苏丹港与科威特市几乎打平，但机制不同

苏丹港年平均 `T2M` 为 `29.43°C`，显著高于科威特市。但苏丹港高温尾部弱得多：

- 苏丹港：P99 `37.30°C`，`>=40°C` 小时 `12`
- 科威特市：P99 `44.09°C`，`>=40°C` 小时 `7828`

所以按 10°C 寿命减半规则指数加权后：

- 科威特市 `T_eq = 29.93°C`
- 苏丹港 `T_eq = 29.92°C`

两者几乎一致，但原因完全不同：苏丹港靠持续温热，科威特市靠夏季高温尾部。

## 5. 工程使用建议

如果目的是做电子器件、风扇、电源、户外设备的可靠性温度样本筛选，建议分三类选点：

| 用途 | 推荐点位 | 原因 |
|---|---|---|
| 长期高均温、全年热暴露 | 吉赞、荷台达、阿萨布、苏丹港 | 冬季冷却时间少，适合看持续热老化 |
| 极端高温尾部、Arrhenius 指数损伤 | 阿瓦士、巴士拉、雅各布阿巴德、科威特市 | `>=45°C` 小时多，适合看高温加速损伤 |
| 中东/海湾典型商用部署代表 | 科威特市、多哈、迪拜、利雅得 | 工程部署常见，但不一定是最严苛 |

建议材料中的保守表述：

> 若按电子可靠性中常见的 Arrhenius 温度加速近似进行小时级指数加权，科威特市并非全球候选热城中最严苛点位。阿瓦士、雅各布阿巴德、巴士拉、吉达、加奥、吉赞等点位在 `2016-2025` NASA POWER `T2M` 数据下的等效温度损伤高于科威特市。吉赞代表全年持续高温暴露，阿瓦士/巴士拉代表极端高温尾部暴露，二者适合分别作为不同可靠性压力场景。

## 6. 边界与待确认事项

本报告结论仅针对“环境温度驱动的 Arrhenius 简化损伤指数”。

需要注意：

- `T2M` 是 2 米环境气温，不是器件结温 `Tj`。
- 实际寿命应使用 `Tj = Tamb + DeltaT`，且 `DeltaT` 可能随太阳辐照、风速、负载、散热结构变化。
- 盐雾、湿度、沙尘、太阳辐照、温度循环、启停、电压应力等未纳入本指标。
- 本次候选点位不是全球城市穷举，不能声明“全球绝对第一”；可以声明“在本次候选热城对照中”。
- 不同失效机制的 `Ea` 不同；“10°C 减半”只适合作为工程初筛。
